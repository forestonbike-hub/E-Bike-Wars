## Referral Loops

**System Architecture:**
```
User gets value → Motivated to share value with others → Shares with friend →
Friend experiences value → Friend becomes user → Cycle repeats
```

**The Closing Mechanism:** The reward for sharing must connect back to getting MORE of what made them want to share. The loop closes when the act of referring directly enhances the referrer's experience.

**Engineering the Loop:**
- **Trigger:** User hits moment of delight or realizes they want more
- **Action:** Invitation feels like giving a gift, not asking a favor
- **Output:** New user enters at same high-value moment
- **Reinvestment:** Referrer gets enhanced experience, priming next referral

**Where to Embed:**
- Immediately after "aha moment" when enthusiasm peaks
- When user hits a natural limit they want to exceed
- At achievement/milestone moments worth celebrating

**What Makes It Circular:**
The key is that sharing isn't separate from using—it IS using. Dropbox's genius: storage is both the product AND the reward. Using more storage creates need for more storage, which motivates more referrals.

## Content/SEO Loops

**System Architecture:**
```
User creates content → Content becomes discoverable asset →
New people discover content → Discoverers become creators → More content created
```

**The Closing Mechanism:** User-generated content must serve dual purposes: value for creator AND discovery magnet for new users. Each piece of content is a door into the product.

**Engineering the Loop:**
- **Trigger:** User wants to create/organize/express something
- **Action:** Creation process naturally produces indexable artifacts
- **Output:** Public content that ranks for long-tail searches
- **Reinvestment:** Discoverers see value, sign up, create more content

**Where to Embed:**
- Make content public by default (with clear value for user)
- Structure content for search discoverability
- Design clear path from "viewer" to "creator"

**What Makes It Circular:**
Every user action builds the discovery engine. Pinterest boards aren't just organization for the user—they're landing pages. The loop compounds because more content = more search surface area = more discoverers = more creators = more content.

## Viral/Frictionless Loops

**System Architecture:**
```
User uses product → Usage naturally involves non-users →
Non-users experience full value → Non-users convert → They involve more non-users
```

**The Closing Mechanism:** The product's core function requires or naturally includes people who aren't yet users, and those people experience the complete value proposition before any commitment.

**Engineering the Loop:**
- **Trigger:** User needs to accomplish something involving others
- **Action:** Others participate without barriers (no account, no download)
- **Output:** Non-users have the "aha moment" pre-signup
- **Reinvestment:** Converted users bring their own networks into the loop

**Where to Embed:**
- Remove every friction point for the invited party
- Ensure invited party experiences FULL value, not teaser
- Place conversion opportunity at moment of peak satisfaction

**What Makes It Circular:**
Zoom's loop: you can't have a meeting alone. Every host necessarily exposes guests to the product. Guests experience everything—no gates, no limits. The conversion happens because they want to HOST, bringing their own guests into the loop.

## Product-Led (Bottom-Up) Loops

**System Architecture:**
```
Individual adopts → Gets value → Brings in colleagues → Team value exceeds individual value →
Team spreads to other teams → Organization-wide adoption → Members carry to new orgs
```

**The Closing Mechanism:** The product works alone but works DRAMATICALLY better together. The gap between solo and team value creates natural expansion pressure.

**Engineering the Loop:**
- **Trigger:** Individual discovers tool, uses it successfully
- **Action:** Realizes team usage would multiply value
- **Output:** Team adoption creates visible success
- **Reinvestment:** Team members become advocates, spread to other teams and eventually other companies

**Where to Embed:**
- Design for immediate individual value (no "you need a team" blocks)
- Make team features visibly better than solo features
- Create natural "moments of invitation" in workflow

**What Makes It Circular:**
Slack works alone (personal notes, reminders). But communication tools are inherently multiplayer. One person using Slack wants to pull in others because that's when it becomes powerful. Those others become advocates who spread it further.

## Marketplace Loops

**System Architecture:**
```
Supply joins → Creates selection/options → Attracts demand →
Demand success attracts more supply → Better selection attracts more demand →
Demand converts to supply
```

**The Closing Mechanism:** Both sides of the marketplace benefit from the other side's growth, AND users can switch sides. Travelers become hosts. Buyers become sellers.

**Engineering the Loop:**
- **Trigger:** One side sees opportunity (earn money, find options)
- **Action:** Participation creates value for the other side
- **Output:** Both sides grow, creating more value for both
- **Reinvestment:** Users switch sides, accelerating both supply and demand

**Where to Embed:**
- Design for side-switching (make it easy to go from consumer to provider)
- Each listing/offering should be discoverable independently
- Success on one side should prompt offer to try the other

**What Makes It Circular:**
Airbnb travelers who have great stays in desirable locations realize: "I could host too." They switch sides, adding supply exactly where demand exists. Their hosting success funds more travel, which exposes them to new hosting ideas.

## Usage-Based Loops

**System Architecture:**
```
User uses product → Output goes to non-user → Non-user sees product in action →
Non-user signs up → Creates outputs that reach more non-users
```

**The Closing Mechanism:** Normal product usage inherently creates artifacts that reach people outside the product, and those artifacts demonstrate value without explanation.

**Engineering the Loop:**
- **Trigger:** User needs to create/send/share something
- **Action:** Output carries product with it (link, embed, branding)
- **Output:** Recipients experience value through the artifact
- **Reinvestment:** Recipients become users, create their own artifacts

**Where to Embed:**
- Every output should be a product demonstration
- Recipients should be able to act on the output, not just view it
- Conversion path embedded in the artifact itself

**What Makes It Circular:**
Calendly: you send a scheduling link. Recipient doesn't see "Calendly"—they see "finally, easy scheduling." They book effortlessly. They think "I need this for MY scheduling." They sign up. They send links. The product IS the marketing.

## Collaboration Loops

**System Architecture:**
```
Creator starts project → Needs input/collaboration → Invites others →
Collaborators see tool value → Collaborators start own projects → They invite others
```

**The Closing Mechanism:** The product's core value requires multiple people. You literally cannot get full value without bringing others in.

**Engineering the Loop:**
- **Trigger:** User wants to create something
- **Action:** Creation process requires or improves with collaboration
- **Output:** Collaborators experience the tool as participants
- **Reinvestment:** Collaborators become creators with their own collaborators

**Where to Embed:**
- Make collaboration integral, not optional
- Allow collaboration with minimal commitment (view/comment without full account)
- Make the jump from collaborator to creator seamless

**What Makes It Circular:**
Figma: a designer creates a mockup. Needs feedback. Shares link. Reviewers comment directly on the design—they're using Figma. When they have their own design needs, Figma is the obvious choice. They share with their collaborators.

## Social Proof Loops

**System Architecture:**
```
User achieves/creates something → Wants to share achievement →
Shares to social network → Network sees and wants similar achievement →
They join → They achieve → They share
```

**The Closing Mechanism:** The product creates moments users WANT to broadcast. Sharing isn't marketing—it's self-expression. The share itself demonstrates product value.

**Engineering the Loop:**
- **Trigger:** User reaches meaningful milestone or creates something share-worthy
- **Action:** Sharing feels like showing off, not promoting
- **Output:** Social posts that make others curious/envious
- **Reinvestment:** New users seek their own shareable moments

**Where to Embed:**
- Create genuine moments of achievement/delight
- Make sharing feel like bragging about yourself, not the product
- Design shareable artifacts that are visually distinctive

**What Makes It Circular:**
Spotify Wrapped: users share because it's about THEM (their music taste, their year). But every share is a Spotify ad. Friends see the posts, wonder about their own stats, sign up to see next year's Wrapped. The vanity IS the virality.

## Choosing the Right Loop Type

**Match loop type to inherent product characteristics:**

| Product Characteristic | Best Loop Types |
|------------------------|-----------------|
| Output goes to non-users | Usage-Based, Viral/Frictionless |
| Multi-user workflows | Collaboration, Product-Led |
| User-generated content | Content/SEO, Social Proof |
| Two-sided transactions | Marketplace |
| Strong emotional moments | Social Proof, Referral |
| Works solo, better together | Product-Led, Collaboration |

**Questions to identify loop opportunities:**
1. What do users create/produce that others see?
2. Who do users naturally involve in their usage?
3. When do users feel proudest or most delighted?
4. What would make the product more valuable if more people used it?
5. How does someone outside the product currently encounter it?

**Layering loops:** Most successful products run 2-3 loops simultaneously. Start with the most natural one (closest to core product behavior), get it working, then layer others.
