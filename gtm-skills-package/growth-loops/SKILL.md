# Growth Loops

Engineer circular, self-feeding systems where growth is built into the product itself—not bolted on as marketing.

## Core Principle

A growth loop is a closed system where **using the product creates more users**. The output of user activity becomes the input for new user acquisition. This creates compounding growth rather than linear growth.

```
User gets value → Action creates exposure → Non-users experience value →
Non-users convert → New users get value → Action creates exposure → ...
```

## Workflow

### 1. Understand the Concept
Before designing loops, deeply understand:
- What does the user actually DO with this?
- Who else is involved when they use it?
- What artifacts/outputs does usage create?
- When is the moment of peak value/delight?

### 2. Identify Loop Opportunities
Ask these questions to find natural loop insertion points:

| Question | Loop Signal |
|----------|-------------|
| Does usage involve non-users? | Viral/Usage-Based potential |
| Does it create shareable outputs? | Content/Social Proof potential |
| Is it better with more people? | Collaboration/Network potential |
| Can consumers become creators? | Marketplace/UGC potential |
| Is there an "aha moment" worth sharing? | Referral/Viral potential |

### 3. Design the Loop
Use this template:

```
TRIGGER: When user [state/need] _______________
ACTION: They [behavior] _______________ which
OUTPUT: Creates [artifact/exposure] _______________ that
REACHES: [audience] _______________ who
CONVERTS: Because they experience [value] _______________
REINVESTS: And immediately [enters trigger state] _______________
```

**Validate the design:**
- Does the action happen naturally during usage, or is it extra work?
- Can non-users experience value BEFORE committing?
- Does the loop actually close—do new users complete the cycle?
- Is the cycle time short (days, not months)?

### 4. Optimize the Mechanics
For each loop, optimize:
- **Exposure:** More non-users reached per user action
- **Conversion:** Higher % of exposed who become users
- **Cycle time:** Faster path from new user to loop completion
- **Trigger strength:** More natural/frequent trigger occurrence

## Loop Types Quick Reference

| Type | Mechanism | Best When |
|------|-----------|-----------|
| **Referral** | Sharing creates mutual benefit | Value can be shared |
| **Content/SEO** | User content becomes discovery engine | Users create public content |
| **Viral/Frictionless** | Usage requires involving non-users | Product involves others |
| **Product-Led** | Individual use → team adoption | Better together |
| **Marketplace** | Supply attracts demand and vice versa | Two-sided transactions |
| **Usage-Based** | Outputs carry product to recipients | External-facing artifacts |
| **Collaboration** | Core value requires multiple users | Team workflows |
| **Social Proof** | Achievement/creation drives sharing | Pride-worthy moments |

## Output Format

When adding growth loops to a concept, deliver:

1. **Concept Analysis:** What the thing is, who uses it, how usage works
2. **Loop Opportunities:** 2-3 potential loop types with rationale
3. **Recommended Loop:** Primary loop to implement with full design:
   - Trigger → Action → Output → Conversion → Reinvestment
   - Why this loop fits this concept
   - Where in the product/feature to embed it
4. **Implementation Details:** Specific changes needed to enable the loop
5. **Success Indicators:** How to know the loop is working

## Anti-Patterns to Avoid

- **Disconnected incentives:** Rewards unrelated to product value
- **Extra steps:** Requiring action outside normal usage
- **Gated experience:** Hiding value behind signup
- **One-time loops:** No sustained motivation to continue
- **Forced sharing:** "Share with friends!" buttons nobody clicks

Growth should be embedded in natural behavior, not layered on as marketing.
